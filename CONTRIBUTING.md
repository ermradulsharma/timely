# Contributing to Timely

Thank you for your interest in contributing to Timely! We welcome contributions from everyone.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:

```bash
git clone https://github.com/your-username/timely.git
cd timely
```

3. **Set up the development environment**:

- Follow the installation steps in [README.md](file:///d:/gitlab/timely/README.md).

4. **Create a new branch** for your feature or bug fix:

```bash
git checkout -b feature/your-feature-name
```

## Coding Standards

- We follow the [PSR-12](https://www.php-fig.org/psr/psr-12/) coding standard.
- Ensure your code is well-documented and includes tests where applicable.
- Run Laravel's built-in testing suite before submitting:
  ```bash
  php artisan test
  ```

## Submitting Pull Requests

1. **Commit your changes** with clear and descriptive messages.
2. **Push your branch** to your fork:

```bash
git push origin feature/your-feature-name
```

3. **Open a Pull Request** against the `main` branch.
4. **Fill out the PR template** completely.

## Bug Reports & Feature Requests

Please use the [issue templates](file:///d:/gitlab/timely/.github/ISSUE_TEMPLATE/) to report bugs or suggest new features. Provide as much detail as possible to help us understand and resolve the issue.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](file:///d:/gitlab/timely/CODE_OF_CONDUCT.md).
